engineering Requirements

Допущения и Non-Functional Requirements

assistant_navigation Принятые допущения

  • «Опоздание на день» = 1 сек – 24 часа после дедлайна
  • Дедлайн сравнивается по серверному времени UTC
  • Опоздание > 24ч → запись с 0 баллов (прозрачный аудит)
  • В лидерборде — текущий баланс баллов
  • Push = web push / существующий канал платформы
  • Audit log обязателен, история начислений — не в MVP

speed Non-Functional Requirements

Баланс API ≤ 200 мс
Лидерборд (c кэшем) ≤ 500 мс
Кэш лидерборда до 1 часа
Масштабируемость 10 000 резидентов
Безопасность Авторизация + Роли
